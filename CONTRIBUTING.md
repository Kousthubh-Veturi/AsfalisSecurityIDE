# Contributing to Asfalis Security Scanner

Thank you for your interest in contributing to Asfalis Security Scanner! This document provides guidelines for contributing to the project.

## Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone. Please be respectful in all interactions.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/asfalis-security.git`
3. Install dependencies: `npm run install:all`
4. Set up your development environment (see SETUP.md)

## Development Workflow

1. Create a feature branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Write or update tests as needed
4. Run tests: `npm test`
5. Commit your changes using conventional commit format
6. Push to your fork and create a pull request

## Commit Message Format

We use conventional commits to maintain a clean history:

```
type(scope): description

body

footer
```

Types:
- `feat`: New features
- `fix`: Bug fixes
- `docs`: Documentation changes
- `style`: Code style changes
- `refactor`: Code refactoring
- `test`: Test additions/changes
- `chore`: Maintenance tasks

## Pull Request Process

1. Ensure your code follows the existing style
2. Update documentation if needed
3. Add tests for new functionality
4. Ensure all tests pass
5. Request review from maintainers

## Security Issues

Please report security vulnerabilities privately to security@asfalis.com rather than creating public issues.

## Questions?

Feel free to open an issue for questions or discussion!
